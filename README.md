# 🏗 Журнал работ на строительном объекте

Веб-приложение для учёта выполненных строительных работ.  
Позволяет вести журнал работ, редактировать записи, фильтровать их и хранить данные в базе данных.

---

## Функциональность

### Основной функционал (по ТЗ)

- Добавление записи о выполненной работе
- Просмотр списка работ
- Удаление записи
- Фильтрация по диапазону дат
- Фильтр по исполнителю
- Выбор вида работ из справочника
- Выбор единицы измерения из списка
- Хранение данных в PostgreSQL

---

### Дополнительные улучшения

- Редактирование записей
- Toast уведомления (успех / ошибки)
- Loading state при загрузке данных
- Empty state при отсутствии записей
- Confirm перед удалением записи
- Sticky header таблицы
- Улучшенный UI и кнопки
- Адаптивная верстка

## Запуск проекта

### 1. Клонирование репозитория

git clone <repository-url>  
cd construction-journal

---

### 2. Запуск базы данных (Docker)

docker compose up -d

---

### 3. Backend

cd backend  
npm install  
npx prisma generate  
npm run dev  

Backend доступен по адресу:  
http://localhost:5000

---

### 4. Frontend

cd frontend  
npm install  
npm run dev  

Frontend доступен по адресу:  
http://localhost:5173

---

## API Endpoints

GET /logs — получить все записи  
POST /logs — создать запись  
PUT /logs/:id — обновить запись  
DELETE /logs/:id — удалить запись  

---

## Архитектура проекта

- React frontend отправляет HTTP запросы через Axios
- Express backend обрабатывает REST API
- Prisma используется как ORM
- PostgreSQL работает в Docker контейнере
- Полная связка frontend ↔ backend ↔ database
